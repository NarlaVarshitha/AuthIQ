AuthIQ API Reference

Welcome to the AuthIQ Developer Portal. Our RESTful API provides real-time digital trust intelligence, enabling platforms and businesses to protect their users from fraudulent e-commerce activities.

Core Features

  • DCTI Scoring: Real-time 0-100 score based on 50+ trust parameters.
  • Risk Assessment: Instant classification (Low, Medium, High, Critical).
  • Scam Reporting: Crowdsourced fraud intelligence database.
  • Merchant Verification: Programmatic access to verified merchant statuses.

API Base URL: /api

Authentication

AuthIQ uses API keys to authenticate your requests. You can generate and manage your unique keys in the Seller Dashboard.

To authenticate, include your API key in the X-AuthIQ-Key HTTP header for every request.

Header Configuration

X-AuthIQ-Key: your_live_key_abcdef12345

Security Tip: Never expose your API keys in client-side code for production. Use a backend proxy to keep your keys secure.

Trust Endpoint

GET /api/trust

Evaluates a specific domain or product URL and returns a comprehensive Trust Report.

Query Parameters

Parameter Type Description
url REQUIRED string The URL to evaluate (e.g., https://example-shop.com)
deep_scan boolean Performs active crawler analysis. Defaults to false.

Success Response

{
  "status": "success",
  "data": {
    "url": "https://example-shop.com",
    "dcti_score": 82,
    "risk_level": "LOW",
    "meta": {
      "scanned_at": "2026-05-06T17:25:00Z",
      "version": "v1.4.2"
    },
    "breakdown": {
      "ssl_security": 95,
      "domain_age": 90,
      "policy_transparency": 70,
      "review_authenticity": 92
    },
    "flags": []
  }
}

Report Scam Endpoint

POST /api/report-scam

Allows developers to contribute to the AuthIQ Trust Network by reporting suspicious sites or links.

Request Body

Field Type Description
domain REQUIRED string The suspicious domain or full URL being reported.
report_type REQUIRED enum One of: phishing, counterfeit, delivery_fraud.
evidence object Supporting data like screenshots, UPI IDs, or phone numbers.

Sample Request

{
  "domain": "scammy-link.biz",
  "report_type": "phishing",
  "evidence": {
    "upi_handle": "baduser@okaxis",
    "phone": "+91 9876543210",
    "description": "Received suspicious payment link via SMS."
  }
}

JavaScript Integration

Integrate AuthIQ trust checks directly into your web applications using the fetch API or your favorite HTTP client.

// Example: Check trust before redirecting to external link
async function checkTrust(targetUrl) {
  const response = await fetch(`https://api.authiq.com/v1/trust?url=${targetUrl}`, {
    headers: { 'X-AuthIQ-Key': 'sk_test_..." }
  });
  
  const result = await response.json();
  
  if (result.data.dcti_score < 40) {
    console.warn("High risk site detected!");
    // Trigger UI warning modal
  }
  
  return result.data;
}

Error Codes

AuthIQ uses standard HTTP response codes to indicate the success or failure of an API request.

Code Meaning Description
200 OK Request was successful.
401 Unauthorized API key is missing or invalid.
429 Too Many Requests Rate limit exceeded for your tier.
500 Server Error An unexpected error occurred on our end.
